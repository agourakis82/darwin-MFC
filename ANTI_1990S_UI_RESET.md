# Anti-1990s UI Reset

## Problem

The PS interface still carries the wrong mental model:

- too many bordered containers
- too many co-equal panels
- too much dashboard language
- too little dominance of the active task
- too much UI explaining itself instead of driving action

This makes the product feel like an old admin portal, not an acute-care app.

## What must stop

### 1. Card inflation
- not every concept deserves a card
- metrics, labels, and helper context should not become standalone surfaces by default

### 2. Equal-weight columns
- left navigation, center task, and right safety cannot look equally important
- the center must clearly win

### 3. Decorative metrics
- numbers that do not change the next action should not dominate
- summary must serve action, not decoration

### 4. Panel stacking
- the user should not feel trapped inside nested panels
- each screen needs a dominant reading path

### 5. Site-like chrome
- header must feel like app chrome
- cockpit must feel like launcher
- overlays must feel like primary work surfaces

## Target interaction model

### Cockpit
- one dominant hero
- three launchable acute workflows
- one compact live-state rail
- one shortcuts cluster

### Runner
- left: dry flow map
- center: dominant active task
- right: compact safety support

### Overlays
- drug sheet: operational review surface
- handoff: transfer surface
- debrief: closure surface

## Visual laws

1. One hero per screen
2. One dominant action cluster per screen
3. Sidebars are subordinate
4. Labels are quiet
5. Overlays are cleaner than pages
6. Motion only reinforces continuity

## Product rule

The Darwin ER PS must feel like:
- a clinical app shell
- a case runner
- a decision workspace

It must not feel like:
- a website
- a dashboard portal
- a card catalog
